# v0.2.14 Risk Reduction Plan

This maintenance note is the tracked handoff record for the real v0.2.14 work. It replaces the withdrawn local-only v0.2.14 authenticator patch record.

## Baseline

- Current release truth: `v0.2.13` is the latest published tag.
- The mistakenly created `v0.2.14` tag and GitHub Release were deleted before this plan began.
- The v0.2.13 replacement state already contains the profile authenticator setup verification, display-state, and temporary profile-status fixes.
- Work branch: `codex/v0.2.14-risk-reduction`.

## Public Contracts

The v0.2.14 work is internal maintenance. Keep these surfaces compatible unless a migration note and compatibility layer are added first:

- settings keys, post meta keys, AJAX action names, nonce names, request fields, and JSON response shapes.
- documented hooks and filters in `docs/hooks.md`.
- template paths, virtual page slugs, public URLs, DOM IDs/classes/data attributes, and modal anchors.
- script/style handles, `window.REIMU_CONFIG`, `window.ReimuWP`, `window.ReimuCommentsRuntime`, and classic script loading.

## Stage Gates

Every high-risk stage must pass:

- `npm run check:comments-profile`
- `npm run check:pjax-runtime`
- `npm run check:js`
- `npm run build`
- `npm run check:size`
- PHP syntax lint for changed PHP files.

Before handoff or release prep, also run:

- `npm run check`
- `npm audit --audit-level=moderate`
- `composer run lint:php`
- full `php -l` over theme PHP files
- `npm run package`
- `npm run check:package`
- `npm run report:php-complexity`

## Manual QA Matrix

Record WordPress manual QA after each high-risk stage:

- Guest auth: invalid login, 2FA-required login, registration code, registration, lost-password code, lost-password reset, GitHub popup trigger.
- Logged-in profile: payload refresh, basic profile save, avatar URL/file, avatar frame, email code, TOTP generate/enable/disable, comment tags, profile status display.
- Comments: submit, reply, empty guard, image/GIF upload, replace/discard upload, like/unlike, edit, delete, sorting, load more.
- Review status: avatar/tag/comment/upload pending, approve, reject, and lightweight status polling.
- PJAX: navigate between comment pages, reopen modals, submit after navigation, use comment tools, verify no duplicate handlers or polling loops.

## Release Rule

Do not create or push `v0.2.14` until automated checks, package validation, and the manual QA matrix are recorded for the final candidate.
