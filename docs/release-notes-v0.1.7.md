# Yneko-Reimu v0.1.7

## Security

- Fixed a GitHub OAuth account-linking risk where starting GitHub login while already signed in as a WordPress administrator could bind the GitHub account to that administrator account.
- Normal GitHub login now only signs in an existing linked GitHub user or creates a new subscriber user when auto-create is enabled. It no longer binds to the current WordPress session.
- GitHub account binding now requires the explicit binding URL generated by `yneko_reimu_github_login_bind_url()` and a valid nonce.
- Prevented a GitHub account that is already linked to one WordPress user from being rebound to another user through the binding flow.

## Highlights

- Hidden the WordPress admin bar on the front end for all logged-in users, including administrators.
- Administrators can still enter the dashboard by visiting `/wp-admin/` directly.
- Non-administrator users are redirected away from `/wp-admin/`, while front-end comment features and AJAX requests remain available.

## Fixes

- Kept GitHub OAuth login and GitHub account binding as separate flows so comment-only users cannot accidentally affect privileged WordPress accounts.
- Preserved subscriber-only behavior for newly auto-created GitHub login users.

## Notes

- Do not continue using v0.1.6 or earlier. Those versions may allow accidental administrator account binding when GitHub login is started from an already-authenticated administrator browser session.
- If a GitHub account was accidentally bound to an administrator account, remove the `_yneko_reimu_github_*` and legacy `_yneko_github_*` user meta values from that administrator before testing again.
- No database migration is required for clean installations.
